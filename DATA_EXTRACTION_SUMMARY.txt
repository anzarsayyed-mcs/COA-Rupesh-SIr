JSON DATA STRUCTURE
================================================================================

Location: data/rm-products.json (3 products)
Location: data/fg-products.json (11 products)

Each Product Contains:
- code: Product code (e.g., "RM2506001", "F2504001")
- name: Short product name (e.g., "Acetone EP/BP")
- productName: Full descriptive name
- category: "raw-material" or "finish-goods"
- standard: Quality standard (NF, EP/BP, HPLC, ND, etc.)
- tests: Array of test specifications

Each Test Contains:
- test: Exact test name from document
- specification: Exact specification text from document
- type: "compliance", "numeric", or "header"
- For compliance: required value ("Complies" or "Passes test")
- For numeric: range (min/max), unit
- For detectables: "detectable" field ("Not Detected")

================================================================================
VALIDATION RULES IMPLEMENTED
================================================================================

Compliance Tests:
- Must select "Complies" or "Passes test" from dropdown
- Exact match required
- Red highlighting if doesn't match

Numeric Tests:
- NLT XX = value must be >= XX (minimum)
- NMT XX = value must be <= XX (maximum)
- Range (min-max) = value must be within range
- Max XX = value must be <= XX
- Green if within spec, Red if out of spec

Detectable Tests:
- Can enter numeric value OR "Not Detected"
- "Not Detected" always passes
- Numeric values validated against range
